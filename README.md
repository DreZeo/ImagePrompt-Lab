# ImagePrompt Lab

ImagePrompt Lab 是一个面向图像生成与提示词工作流的本地优先 Web 工具。它基于 OpenAI 兼容的图片接口，提供文本生图、参考图编辑、历史任务管理、预设提示词库、自定义模板和 AI 提示词助手，适合个人创作、商品视觉、社媒内容、UI 截图、信息图和多轮图像迭代。

本项目基于 [CookSleep/gpt_image_playground](https://github.com/CookSleep/gpt_image_playground) 二次开发，感谢原项目提供的基础能力与开源工作。

---

## 功能特性

### 图像生成与编辑

- **文本生图**：输入提示词后调用图片生成接口创建图像。
- **参考图编辑**：支持上传、粘贴或拖拽参考图，并基于参考图继续编辑生成。
- **批量生成**：可按当前参数一次生成多张结果。
- **多轮迭代**：可将生成结果加入参考图列表，继续进行下一轮编辑。

### 参数与工作流

- **尺寸与比例控制**：支持常用比例、分辨率档位和自定义尺寸。
- **输出格式**：支持 PNG、JPEG、WebP 等格式及压缩质量配置。
- **任务历史**：记录提示词、参数、参考图、结果图和耗时，方便回看与复用。
- **导入导出**：可将本地任务和图片资源打包为 ZIP，便于备份和迁移。

### 提示词与预设

- **AI 助手**：根据用户意图提取视觉需求，召回本地风格、模板和规则，辅助生成更稳定的提示词。
- **结构化模板**：内置多类官方模板，覆盖海报、人像、产品、UI、信息图、场景和通用设计。
- **自定义模板**：支持创建、编辑和保存自己的结构化提示词模板，并参与 AI 检索。
- **预设库治理**：区分官方模板、旧版模板和用户模板，保留本地可控的提示词资产。

### 本地优先

- **浏览器本地存储**：任务数据和图片资源主要保存在 IndexedDB 中。
- **本地设置**：API 配置、生成参数等通过浏览器本地状态保存。
- **无后端依赖**：默认作为纯前端应用运行，可直接连接兼容接口。

## 快速开始

### 环境要求

- Node.js 18 或更高版本
- npm

### 安装依赖

```bash
npm install
```

### 启动开发服务

```bash
npm run dev
```

启动后访问 Vite 输出的本地地址，通常是：

```text
http://localhost:5173
```

### 构建生产版本

```bash
npm run build
```

### 本地预览构建结果

```bash
npm run preview
```

## API 配置

打开页面右上角设置按钮后，可以分别配置：

- **图片生成 API**：用于文本生图和参考图编辑。
- **Chat API**：用于 AI 提示词助手、模型获取和连接测试。

图片 API 与 Chat API 的地址、Key 和模型配置彼此独立。项目支持 OpenAI 兼容接口，实际可用模型与能力取决于你配置的服务商。

如需在构建时设置默认图片 API 地址，可在项目根目录创建 `.env.local`：

```bash
VITE_DEFAULT_API_URL=https://api.openai.com
```

## 数据与隐私

- 任务历史、图片资源和部分设置保存在当前浏览器本地。
- 导出 ZIP 会包含任务清单、图片文件和必要元数据。
- 更换浏览器、清理站点数据或改动本地存储 key 可能导致旧设置不可见。
- API 请求会发送到你在设置中配置的接口服务，请自行确认服务商的数据策略。
